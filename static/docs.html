From zero to deployed

Four commands. No accounts. No clouds. Just types.

  1. 1

    Install

    cargo install --git https://github.com/thepictishbeast/PlausiDen-Forge --locked --tag v0.18 forge-cli

  2. 2

    Author

    Drop one JSON file per page in cms/. Editor autocomplete via the $schema reference; closed-enum CmsSection variants enforce shape at parse time.

  3. 3

    Build

    forge build runs 27 audit phases + emits a Merkle-chained Ed25519-signed report. Output: static/ + reports/.

  4. 4

    Deploy

    rsync static/ to any host. Caddy, nginx, IPFS, Tor hidden service, your laptop. No platform lock-in.

Reference

The complete documentation lives in the source tree.

Section kinds
Closed-enum CmsSection variants spanning Structural, Editorial, Marketing, Media, Commerce, Social, Forms, Navigation, Feedback, Game-Forum-Video, Auth, and Crucible families. Every kind carries deny_unknown_fields; a typo at author time is a parse failure, not a render-time blank.
Themes
AMOLED true-black, WCAG-AA high-contrast pair, sepia, warm, ocean, forest, violet, rose. Picked via data-theme attribute on <html> or via the in-page toggle. Same OKLCH palette; no theme uses a different color scale.
Audit phases
Configured in forge.toml. Tokens / CSP / a11y landmarks / theme contrast / phantom-button / unbuilt-route / external-assets / image-desert / heading-order / contrast-AA / and more. Each phase is a typed Rust struct; severities are info / warn / strict.

Common questions

Where's the schema?

cms-schema.json at your project root. Reference it via "$schema": "../cms-schema.json" in every cms/*.json. Editors with jsonls (VS Code / Zed / Helix / Neovim) provide inline autocomplete + validation.

How do I add a new primitive?

Loom-side concern: extend CmsSection enum + render_section arm + skin.css block. See the PlausiDen-Loom contribution guide for the pattern.

Can I author with the GUI?

loom edit serve ships a typed-CMS editor that writes back to cms/*.json. Or hand-edit the JSON — both paths are first-class.

How do I host on Tor?

Add a DeployAdapter entry to your deploy.toml with kind = "onion". forge deploy then publishes your build to the hidden service.
