GitHub features.

Five interlocking surfaces that turn a git repo into the substrate every other engineering tool plugs into — code, CI, AI, work tracking, and decision capture share one authz model.

Repositories

Source of truth for code + collaboration. Distributed git hosting with rich permissions, branch protection, code review at the line level. The substrate everything else depends on.

Actions

CI/CD that runs in the same place the code lives. Workflow-as-code: matrix builds, custom runners, secrets management, marketplace integration. Same authz model as the repo itself.

Copilot

AI pair programmer + agent integration. Inline suggestions, agent-mode PRs, code review augmentation. Integrates with the chat surfaces (Discussions, PR comments) where engineering decisions actually happen.

Issues

Typed issue + project tracking. Labels, milestones, projects, custom fields. Tied to commits + PRs + discussions through closes-via-mention semantics.

Discussions

Long-form decision capture. RFC threads, Q&A, announcements. Distinct from issues — the place engineering decisions get litigated before they become work items.
