What's in the box

Six tools. Use one. Or all six.

Forge

Typed build pipeline. 27 audit phases per commit, Merkle-chained signed reports, deterministic output.

Loom

132 typed UI primitives, 13 themes, OKLCH palette, dual-theme by default, WCAG 2.1 AA gated at build time.

Crawler

Runtime auditor walking the real DOM. 120+ typed detectors no static analysis can run.

Annotator

Operator capture. Typed event stream, lossless replay, local-first storage, opt-in by user.

CMS

Content as typed data. JSON files in version control. No database, no template language.

LFI

AI evaluation, not generation. Critic trait gates every proposal. LLM-pluggable, optional.

How a build runs

Every commit traces through the same pipeline.

  1. 1

    Parse CMS

    Read cms/*.json. Closed-enum CmsSection variants with deny_unknown_fields catch malformed input at the boundary.

  2. 2

    Validate

    27 audit phases — tokens, CSP, a11y landmarks, semantic HTML, theme contrast, path consistency, perf budget.

  3. 3

    Render

    loom-cms-render walks the typed page, emits semantic HTML + the Loom skin. No template language.

  4. 4

    Sign + chain

    Build report hashed into the Merkle chain. Ed25519 signature. Independently verifiable.

Platform questions

Can I use only one tool?

Yes. Forge is the build pipeline; everything else is opt-in.

Common starting point: Forge + Loom. Add Crawler when you want runtime audits. Add Crucible + forge-auth when you ship logins.

How does the LFI integration work?

forge-critic ships the Critic trait + NoopCritic + LlmCritic in every build. The LFI-backed Critic is gated behind cargo --features lfi.

Bring your own LLM provider. The platform never sees your prompts.

Does it deploy to Tor / I2P / IPFS?

Yes. Six DeployAdapter implementations: clearnet, Tor onion, I2P eepsite, IPFS/IPNS, Gemini, Lokinet.
